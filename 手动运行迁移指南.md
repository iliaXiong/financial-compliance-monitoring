# 手动运行数据库迁移指南

## 问题

由于 IPv4/IPv6 兼容性问题，无法通过命令行连接到 Supabase 数据库。

## ✅ 解决方案：使用 Supabase SQL Editor

### 步骤 1：打开 SQL Editor

1. 访问 https://supabase.com/dashboard
2. 选择你的项目（Reference ID: `ynbaatdsceqtqwmqhlgu`）
3. 点击左侧菜单的 **SQL Editor** 图标

### 步骤 2：创建新查询

1. 点击 **"New Query"** 按钮
2. 会打开一个空白的 SQL 编辑器

### 步骤 3：复制 SQL 脚本

打开项目中的文件：`完整数据库迁移.sql`

或者运行以下命令查看内容：
```bash
cat 完整数据库迁移.sql
```

### 步骤 4：粘贴并执行

1. 将 `完整数据库迁移.sql` 的全部内容复制
2. 粘贴到 Supabase SQL Editor 中
3. 点击右下角的 **"Run"** 按钮（或按 Cmd/Ctrl + Enter）

### 步骤 5：验证表已创建

1. 点击左侧菜单的 **"Table Editor"** 图标
2. 应该看到以下表：
   - ✅ users
   - ✅ tasks
   - ✅ executions
   - ✅ retrieval_results
   - ✅ original_contents
   - ✅ summary_documents
   - ✅ comparison_reports
   - ✅ cross_site_analyses

### 部署到 Render

1. 访问 https://dashboard.render.com
2. 点击 **"New +"** → **"Web Service"**
3. 连接你的 GitHub 仓库
4. 配置：
   ```
   Name: financial-compliance-backend
   Region: Singapore
   Branch: main
   Root Directory: backend
   
   Build Command: npm install && npm run build
   Start Command: node dist/index.pgboss.js
   ```

5. 添加环境变量：
   - 点击 **"Environment"** 标签
   - 复制 `.render-env.txt` 中的所有变量
   - 逐个添加或批量粘贴

6. 点击 **"Create Web Service"**

### 部署前端到 Vercel

等待 Render 部署完成后（获得后端 URL）：

1. 创建前端环境变量：
   \`\`\`bash
   cat > frontend/.env.production << EOF
   VITE_API_URL=https://your-backend-url.onrender.com
   EOF
   \`\`\`
   
   将 `your-backend-url` 替换为实际的 Render URL

2. 部署到 Vercel：
   \`\`\`bash
   cd frontend
   npx vercel --prod

## 📝 注意事项

### 为什么手动运行迁移？

- Supabase 项目使用 IPv6，但本地网络只支持 IPv4
- Connection Pooler 需要特殊配置
- 手动运行是最简单可靠的方法

### Render 部署会正常工作吗？

✅ 会的！因为：
- Render 服务器支持 IPv6
- 使用 Connection Pooler 配置
- 不需要本地连接

### 如果 SQL 执行出错？

1. 检查错误信息
2. 可能某些表已存在，可以忽略 "already exists" 错误
3. 或者先删除已存在的表，再重新运行

删除所有表的 SQL（谨慎使用）：
\`\`\`sql
DROP TABLE IF EXISTS cross_site_analyses CASCADE;
DROP TABLE IF EXISTS comparison_reports CASCADE;
DROP TABLE IF EXISTS summary_documents CASCADE;
DROP TABLE IF EXISTS original_contents CASCADE;
DROP TABLE IF EXISTS retrieval_results CASCADE;
DROP TABLE IF EXISTS executions CASCADE;
DROP TABLE IF EXISTS tasks CASCADE;
DROP TABLE IF EXISTS users CASCADE;
\`\`\`
