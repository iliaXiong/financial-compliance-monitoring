## 🎯 两种部署方式

### 方式 1: 自动化脚本（推荐）⚡

最简单的方式，脚本会引导你完成所有步骤：

```bash
./scripts/deploy-render-supabase-vercel.sh
```

脚本会帮你：
- ✅ 配置 Supabase 数据库
- ✅ 运行数据库迁移
- ✅ 生成 Render 环境变量
- ✅ 生成 Vercel 配置文件

### 方式 2: 手动部署 📝

按照 3 个简单步骤操作：

#### 第 1 步：Supabase (5 分钟)

1. 访问 https://supabase.com，创建项目
2. 获取数据库连接信息
3. 运行迁移：
   ```bash
   export DB_HOST=aws-0-ap-southeast-1.pooler.supabase.com
   export DB_PASSWORD=你的密码
   cd backend && npm install && npm run migrate
   ```

#### 第 2 步：Render (10 分钟)

1. 访问 https://render.com，创建 Web Service
2. 配置：
   - Root Directory: `backend`
   - Build Command: `npm install && npm run build`
   - Start Command: `node dist/index.pgboss.js`
3. 添加环境变量（见下方）
4. 部署

**必需的环境变量**：
```bash
NODE_ENV=production
PORT=3000
DB_HOST=aws-0-ap-southeast-1.pooler.supabase.com
DB_PASSWORD=你的密码
JWT_SECRET=$(openssl rand -base64 32)

# LLM 配置（二选一）
OPENAI_API_KEY=your-key  # 或
LLM_API_URL=your-url
LLM_API_KEY=your-key
```

#### 第 3 步：Vercel (5 分钟)

1. 创建 `frontend/.env.production`：
   ```bash
   VITE_API_BASE_URL=https://your-app.onrender.com
   ```

2. 部署：
   ```bash
   cd frontend
   vercel --prod
   ```

### 常见问题

**Q: 完全免费吗？**
A: 是的！所有服务都使用免费层，总成本 $0/月。

**Q: Render 会休眠吗？**
A: 是的，15 分钟无活动后休眠。使用 UptimeRobot 可以防止休眠。

**Q: 需要 Redis 吗？**
A: 不需要！我们使用 pg-boss（基于 PostgreSQL）作为任务队列。

**Q: 数据会丢失吗？**
A: 不会。Supabase 提供自动备份。
