# 快速诊断清单 - 检索无结果问题

**问题**: 最新的检索仍然没有结果  
**目标**: 快速找出根本原因

---

## 5分钟快速诊断

### ✅ 步骤1: 检查Railway环境变量（1分钟）

访问: https://railway.app/dashboard → 项目 → 后端服务 → Variables

**必须存在的变量**:
- [ ] `DEBUG_MODE=true`
- [ ] `MAX_CHUNKS_PER_KEYWORD=30`
- [ ] `CHUNK_MAX_SIZE=500`
- [ ] `CHUNK_MIN_SIZE=100`
- [ ] `CHUNK_OVERLAP=50`

**如果缺少**: 添加后保存，Railway会自动重新部署（等待3分钟）

---

### ✅ 步骤2: 检查Railway最新部署（1分钟）

Railway Dashboard → Deployments

**检查**:
- [ ] 最新部署时间是否在添加环境变量之后？
- [ ] 部署状态是否为 "Active"（绿色）？
- [ ] 构建是否成功？

**如果不是**: 点击 "Redeploy" 重新部署

---

### ✅ 步骤3: 检查Railway日志（2分钟）

Railway Dashboard → Deployments → 最新部署 → View Logs

**搜索以下关键词**:

#### 3.1 搜索: `Using optimized`
```
期望看到: [ContentRetriever] Using optimized LLM search
```
- ✅ 看到了 → 优化版本已启用，继续下一步
- ❌ 没看到 → **问题**: 代码未部署或未生效

#### 3.2 搜索: `Total chunks created`
```
期望看到: [ContentRetriever] Total chunks created: X (X > 0)
```
- ✅ X > 0 → Chunk拆分正常，继续下一步
- ❌ X = 0 → **问题**: 内容太短或抓取失败

#### 3.3 搜索: `DEBUG INFO`
```
期望看到: ========== DEBUG INFO ==========
```
- ✅ 看到了 → DEBUG模式已开启，继续下一步
- ❌ 没看到 → **问题**: DEBUG_MODE未生效

#### 3.4 搜索: `Error` 或 `error`
```
检查是否有错误信息
```
- ✅ 没有错误 → 继续下一步
- ❌ 有错误 → **记录错误信息**

---

### ✅ 步骤4: 检查数据库（1分钟）

在Supabase SQL Editor运行:

```sql
-- 查看最新的检索结果
SELECT 
    rr.website_url,
    rr.status,
    rr.error_message,
    jsonb_array_length(rr.keyword_matches) as keyword_count,
    rr.retrieved_at
FROM retrieval_results rr
ORDER BY rr.retrieved_at DESC
LIMIT 3;
```

**检查**:
- [ ] 是否有最新的记录？
- [ ] status是什么？
- [ ] keyword_count是多少？
- [ ] error_message是什么？

## 常见问题和解决方案

### 问题A: 没有看到 "Using optimized LLM search"

**原因**: 优化代码未部署

**解决方案**:
1. 确认最新commit包含优化代码
   ```bash
   git log --oneline -1
   # 应该看到: feat: 阶段一RAG优化
   ```
2. 在Railway Dashboard点击 "Redeploy"
3. 等待3分钟后重新检查日志

---

### 问题B: 看到 "Total chunks created: 0"

**原因**: 内容太短或抓取失败

**检查日志**:
```
搜索: "Successfully fetched"
```

**如果看到 "Successfully fetched"**:
- 内容太短（<100字符）
- 解决: 在Railway添加 `CHUNK_MIN_SIZE=50`

**如果没看到 "Successfully fetched"**:
- Jina Reader抓取失败
- 解决: 检查网站URL是否正确，尝试其他网站

---

### 问题C: 没有看到 "DEBUG INFO"

**原因**: DEBUG_MODE未生效

**解决方案**:
1. 在Railway Variables中添加 `DEBUG_MODE=true`
2. 保存后等待自动重新部署（3分钟）
3. 创建新的测试任务
4. 重新检查日志

---

### 问题D: 看到DEBUG INFO但found=false

**原因**: LLM判断关键词不存在

**检查DEBUG INFO中的**:
```json
{
  "retrieval": {
    "retrievedChunks": X  // 检索到的chunks数量
  },
  "llmAnswer": {
    "found": false,
    "definition": ""  // LLM的回答
  }
}
```

**如果 retrievedChunks = 0**:
- BM25没有找到相关内容
- 解决: 使用更相关的关键词

**如果 retrievedChunks > 0 但 found = false**:
- LLM判断关键词不存在
- 解决: 检查关键词是否真的在网站内容中

---

### 问题E: 看到错误信息

**常见错误**:

#### "LLM API key is not configured"
- 解决: 在Railway添加 LLM_API_KEY

#### "Connection timeout"
- 解决: 增加 RETRIEVAL_TIMEOUT_MS

#### "Database connection failed"
- 解决: 检查DB_HOST, DB_PASSWORD
