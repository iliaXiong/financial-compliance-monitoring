## 📋 准备工作

在运行脚本前，请准备：

### 1. 获取正确的 Reference ID

**不要使用项目名称！**

正确方法：
1. 访问 https://supabase.com/dashboard
2. 点击你的项目
3. 点击 **Settings** (左侧齿轮图标)
4. 点击 **General**
5. 找到 **Reference ID**（20 个小写字母）

示例：
- ✅ 正确：`ynbaatdsceqtqwmqhlgu` (20 个字符)
- ❌ 错误：`financial-compliance-monitoring` (项目名称)

### 2. 确认数据库密码

这是你创建 Supabase 项目时设置的密码。

**如果忘记了**：
1. Settings → Database
2. 点击 "Reset database password"
3. 设置新密码

### 3. 确认项目已启动

新项目需要 2-3 分钟才能完全启动。

检查：Dashboard 中项目状态应该是 **Active** (绿色)

## 📦 部署到 Render

迁移成功后：

### 1. 创建 Web Service

1. 访问 https://dashboard.render.com
2. 点击 **New +** → **Web Service**
3. 连接你的 GitHub 仓库

### 2. 配置服务

```
Name: financial-compliance-backend
Region: Singapore
Branch: main
Root Directory: backend

Build Command: npm install && npm run build
Start Command: node dist/index.pgboss.js
```

### 3. 添加环境变量

打开 `.render-env.txt` 文件，复制所有变量到 Render：

- 点击 **Environment** 标签
- 点击 **Add Environment Variable**
- 逐个添加（或批量粘贴）

关键变量：
```
DB_HOST=aws-0-ap-southeast-1.pooler.supabase.com
DB_PORT=6543
DB_USER=postgres.{your_reference_id}
DB_PASSWORD={your_password}
```

### 4. 部署

点击 **Create Web Service**

等待 5-10 分钟完成首次部署。

## 🌐 部署前端到 Vercel

后端部署成功后：

### 1. 创建生产环境配置

```bash
cat > frontend/.env.production << EOF
VITE_API_URL=https://your-backend.onrender.com
EOF
```

将 `your-backend` 替换为你的 Render 服务 URL。

### 2. 部署到 Vercel

```bash
cd frontend
npx vercel --prod
```

或者通过 Vercel Dashboard：
1. 访问 https://vercel.com
2. Import 你的 GitHub 仓库
3. 配置：
   - Framework: Vite
   - Root Directory: frontend
   - Environment Variable: `VITE_API_URL`

### 常见问题

**Q: Reference ID 在哪里？**
A: Settings → General → Reference ID（不是项目名称）

**Q: 项目刚创建，能立即迁移吗？**
A: 不能，需要等待 2-3 分钟让项目完全启动

**Q: 密码忘记了怎么办？**
A: Settings → Database → Reset database password

**Q: 测试脚本显示连接失败？**
A: 检查 Reference ID 和密码，确认项目状态是 Active
