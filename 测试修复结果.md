# 测试修复结果 - 快速指南

## 🎯 目标
验证超时问题已修复，任务可以正常执行并返回结果。

## ⏱️ 预计时间
总共约 10 分钟

## 📋 测试步骤

### 步骤1: 等待Railway部署（3分钟）

1. 打开 https://railway.app
2. 进入项目 "financial-compliance-monitoring"
3. 点击 Backend 服务
4. 查看部署状态：
   - 🔄 "Building" → 正在构建
   - 🔄 "Deploying" → 正在部署
   - ✅ "Success" → 部署成功（绿色）

**等待直到状态变为 "Success"**

### 步骤2: 创建测试任务（1分钟）

1. 打开 https://financial-compliance-monitoring.vercel.app/tasks
2. 点击"创建新任务"
3. 填写表单：
   ```
   任务名称: 超时修复测试
   关键词: professional
   目标网站: https://www.nasdaq.com/
   执行频率: 一次性
   ```
4. 点击"创建任务"
5. 点击"立即执行"

### 步骤3: 观察Railway日志（3-5分钟）

1. 回到Railway Dashboard
2. 在Backend服务页面，点击"Logs"标签
3. 观察日志输出，应该看到：


**关键指标**:
- LLM API响应时间应该在日志中显示
- 不应该看到 "Execution Timeout" 错误
- 应该看到 "completed successfully"

### 步骤4: 检查SQL结果（2分钟）

1. 打开 Supabase Dashboard
2. 进入 SQL Editor
3. 执行以下查询：

```sql
-- 查看最新执行的结果
SELECT 
  e.id as execution_id,
  e.status as execution_status,
  e.error_message,
  EXTRACT(EPOCH FROM (e.end_time - e.start_time)) as duration_seconds,
  r.website_url,
  r.keyword,
  r.found,
  LEFT(COALESCE(r.content, r.context, '(无内容)'), 200) as content_or_error
FROM executions e
LEFT JOIN retrieval_results r ON r.execution_id = e.id
WHERE e.task_id = (
  SELECT id FROM tasks WHERE name = '超时修复测试'
)
ORDER BY e.start_time DESC, r.created_at
LIMIT 10;
```

**预期结果**:

#### 成功情况A: 找到关键词
```
execution_status: completed
error_message: null
duration_seconds: 60-180 (不超过300)
found: true
content_or_error: <关键词的定义和解释>
```

#### 成功情况B: 未找到关键词
```
execution_status: completed
error_message: null
duration_seconds: 60-180 (不超过300)
found: false
content_or_error: (无内容) 或空
```

#### ❌ 失败情况（不应该出现）
```
execution_status: completed
error_message: null
found: false
content_or_error: Error: Execution Timeout - task did not complete in time
```

## ✅ 成功标准

修复成功的标志：

1. ✅ **Railway日志显示**:
   - LLM API被调用
   - LLM API响应时间被记录
   - 执行完成（completed successfully）
   - 没有超时错误

2. ✅ **SQL结果显示**:
   - execution_status = 'completed'
   - error_message = null
   - duration_seconds < 300
   - 没有 "Execution Timeout" 错误

3. ✅ **前端显示**:
   - 任务状态显示为"已完成"
   - 可以查看执行结果
   - 有摘要文档（即使found=false）

## ❌ 如果仍然失败

### 情况1: 仍然超时（duration > 300秒）

**可能原因**:
- LLM API响应极慢（>120秒）
- 网络连接问题

**解决方案**:
1. 查看Railway日志中的LLM API响应时间
2. 如果>120秒，需要进一步增加超时
3. 或者考虑更换LLM API

### 情况2: LLM API调用失败

**Railway日志显示**:
```
[ContentRetriever] Error calling LLM API: <error>
[ContentRetriever] LLM search failed
[ContentRetriever] Falling back to simple keyword matching
```

**可能原因**:
- LLM API密钥无效
- LLM API URL错误
- 网络连接问题

**解决方案**:
1. 检查Railway环境变量
2. 测试LLM API是否可访问
3. 查看详细错误信息

### 情况3: 其他错误

**提供以下信息**:
1. Railway日志（完整的执行日志）
2. SQL查询结果
3. 错误信息
