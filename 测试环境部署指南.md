## 故障排查

### 问题1: 构建失败

**症状**: Render构建失败

**可能原因**:
1. 依赖安装失败
2. TypeScript编译错误
3. 环境变量缺失

**解决方法**:
```bash
# 查看构建日志
# 在Render Dashboard → Logs → Build Logs

# 常见错误：
# 1. natural库安装失败
#    解决: 确保package.json中有natural@8.1.1

# 2. TypeScript编译错误
#    解决: 本地运行npm run build检查

# 3. 内存不足
#    解决: Render免费层有512MB限制，应该足够
```

### 问题2: 启动失败

**症状**: 服务启动后立即崩溃

**可能原因**:
1. 启动命令错误
2. 数据库连接失败
3. 环境变量缺失

**解决方法**:
```bash
# 检查启动命令
# 必须是: node dist/index.pgboss.js

# 检查数据库连接
# 确保DB_HOST, DB_PASSWORD等变量正确

# 检查日志
# 在Render Dashboard → Logs → Runtime Logs
```

### 问题3: Debug信息未显示

**症状**: 日志中没有debug信息

**可能原因**:
1. DEBUG_MODE未设置为true
2. 环境变量未生效

**解决方法**:
```bash
# 1. 检查环境变量
# 在Render Dashboard → Environment
# 确认DEBUG_MODE=true

# 2. 重启服务
# 在Render Dashboard → Manual Deploy → Deploy latest commit

# 3. 查看日志
# 应该看到"DEBUG INFO"字样
```

### 问题4: 性能未改善

**症状**: Token使用量没有降低

**可能原因**:
1. 优化代码未生效
2. 配置参数不正确
3. 旧代码缓存

**解决方法**:
```bash
# 1. 确认使用优化版本
# 查看日志，应该看到：
# [ContentRetriever] Using optimized LLM search

# 2. 检查配置
# MAX_CHUNKS_PER_KEYWORD=30
# CHUNK_MAX_SIZE=500

# 3. 清除缓存重新部署
# 在Render Dashboard → Manual Deploy → Clear build cache & deploy
```
