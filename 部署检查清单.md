# ✅ 部署检查清单

使用此清单确保部署顺利完成。

---

## 📋 部署前准备

- [ ] 已注册 Supabase 账号 (https://supabase.com)
- [ ] 已注册 Render 账号 (https://render.com)
- [ ] 已注册 Vercel 账号 (https://vercel.com)
- [ ] 已将代码推送到 GitHub
- [ ] 已准备好 LLM API 配置（OpenAI 或自定义 API）

---

## 🗄️ 第一步：Supabase 数据库

- [ ] 创建 Supabase 项目
- [ ] 设置数据库密码
- [ ] 获取数据库连接信息（DB_HOST, DB_PASSWORD）
- [ ] 在本地运行数据库迁移
  ```bash
  export DB_HOST=aws-0-ap-southeast-1.pooler.supabase.com
  export DB_PASSWORD=你的密码
  cd backend && npm run migrate
  ```
- [ ] 验证迁移成功（在 Supabase SQL Editor 查看表）

---

## 🖥️ 第二步：Render 后端

- [ ] 创建 Web Service
- [ ] 连接 GitHub 仓库
- [ ] 配置服务设置：
  - [ ] Root Directory: `backend`
  - [ ] Build Command: `npm install && npm run build`
  - [ ] Start Command: `node dist/index.pgboss.js`
  - [ ] Instance Type: Free
- [ ] 添加所有环境变量（见下方清单）
- [ ] 点击 "Create Web Service"
- [ ] 等待部署完成（5-10 分钟）
- [ ] 验证健康检查：`curl https://your-app.onrender.com/health`
- [ ] 记录后端 URL

### 环境变量清单

- [ ] `NODE_ENV=production`
- [ ] `PORT=3000`
- [ ] `TZ=Asia/Shanghai`
- [ ] `DB_HOST`
- [ ] `DB_PORT=5432`
- [ ] `DB_NAME=postgres`
- [ ] `DB_USER=postgres`
- [ ] `DB_PASSWORD`
- [ ] `JWT_SECRET`（使用 `openssl rand -base64 32` 生成）
- [ ] `JWT_EXPIRES_IN=7d`
- [ ] LLM 配置（二选一）：
  - [ ] OpenAI: `OPENAI_API_KEY`, `OPENAI_MODEL`
  - [ ] 自定义: `LLM_API_URL`, `LLM_API_KEY`, `LLM_MODEL`, `LLM_API_KEY_HEADER`, `LLM_AUTH_PREFIX`
- [ ] `JINA_READER_API_URL=https://r.jina.ai`
- [ ] `MAX_PARALLEL_WEBSITES=5`
- [ ] `RETRIEVAL_TIMEOUT_MS=30000`
- [ ] `ENABLE_WEBSITE_ANALYZER=true`
- [ ] `DEMO_MODE=false`

---

## 🎨 第三步：Vercel 前端

- [ ] 创建 `frontend/.env.production` 文件
- [ ] 设置 `VITE_API_BASE_URL=https://your-app.onrender.com`
- [ ] 部署到 Vercel：
  - [ ] 方式 1: 运行 `cd frontend && vercel --prod`
  - [ ] 方式 2: 在 Vercel Dashboard 导入项目
- [ ] 配置 Vercel 项目：
  - [ ] Root Directory: `frontend`
  - [ ] Framework: Vite
  - [ ] Build Command: `npm run build`
  - [ ] Output Directory: `dist`
- [ ] 添加环境变量 `VITE_API_BASE_URL`
- [ ] 等待部署完成（2-5 分钟）
- [ ] 记录前端 URL

---

## 🔍 第五步：监控和验证

- [ ] 后端健康检查正常
  ```bash
  curl https://your-app.onrender.com/health
  ```
- [ ] 查看 Render 日志，无错误
- [ ] 查看 Vercel 日志，无错误
- [ ] 在 Supabase Dashboard 查看数据：
  ```sql
  -- 查看任务
  SELECT * FROM tasks;
  
  -- 查看执行记录
  SELECT * FROM executions;
  
  -- 查看队列任务
  SELECT * FROM pgboss.job ORDER BY createdon DESC LIMIT 10;
  ```
- [ ] 测试定时任务（如果配置了）

## ❓ 遇到问题？

查看 [RENDER_SUPABASE_VERCEL_部署指南.md](./RENDER_SUPABASE_VERCEL_部署指南.md) 的"常见问题"部分。

常见问题：
- Render 休眠问题 → 使用 UptimeRobot
- 前端无法连接后端 → 检查 CORS 和环境变量
- 数据库连接失败 → 检查 Supabase 密码和防火墙
- 任务执行失败 → 查看 Render 日志
