# 完整诊断指南

## 问题描述
任务执行后没有检索到结果，需要诊断原因。

## 诊断步骤

### 步骤1: 在Supabase SQL Editor中执行诊断SQL

打开 `诊断test任务.sql` 文件，复制所有SQL语句到Supabase SQL Editor执行。

### 步骤2: 分析结果

根据SQL执行结果，检查以下几点：

#### 2.1 任务信息
- 任务是否存在？
- 关键词是什么？
- 目标网站是什么？

#### 2.2 执行记录
- 执行是否完成（status = 'completed'）？
- 是否有错误信息（error_message）？
- 执行时长是否正常？

#### 2.3 检索结果详情
**这是最关键的部分！**

检查每个检索结果：
- `found` 字段：是否为 true？
- `content_or_error` 字段：
  - 如果 found=true，应该显示内容的前500字符
  - 如果 found=false，应该显示错误信息（context字段）
- `source_url` 字段：是否正确？
- `document_url` 字段：是否有文档URL？

#### 2.4 检索结果统计
- `total_results`：总共有多少条检索结果？
- `found_count`：找到内容的结果数量
- `not_found_count`：未找到内容的结果数量

#### 2.5 分析结果
- `summary_count`：是否生成了摘要文档？
- `comparison_count`：是否生成了对比报告？
- `cross_site_count`：是否生成了跨站点分析？

## 可能的问题和解决方案

### 问题1: found=false，context显示"Error: ..."
**原因**: 网站访问失败或Jina Reader API失败

**解决方案**:
1. 检查网站是否可访问
2. 检查Jina Reader API是否正常
3. 查看Railway日志中的详细错误信息

### 问题2: found=false，context为空或"No matches found"
**原因**: LLM搜索未找到关键词

**可能的子原因**:
1. **LLM API配置错误**: 检查Railway环境变量
   - `LLM_API_KEY`: API密钥是否正确？
   - `LLM_API_URL`: API URL是否正确？
   - `LLM_MODEL`: 模型名称是否正确？
   - `LLM_API_KEY_HEADER`: 请求头名称（默认: Authorization）
   - `LLM_AUTH_PREFIX`: 认证前缀（默认: Bearer）

2. **LLM响应格式错误**: LLM返回的JSON格式不正确
   - 查看Railway日志中的LLM响应
   - 检查是否有JSON解析错误

3. **关键词在网站中确实不存在**: 关键词可能不在目标网站上

4. **LLM理解错误**: LLM可能误判了关键词的存在

### 问题3: total_results=0
**原因**: 执行流程在保存检索结果之前就失败了

**解决方案**:
1. 查看执行记录的error_message
2. 查看Railway日志中的详细错误堆栈

### 问题4: summary_count=0
**原因**: 摘要生成失败

**可能的子原因**:
1. 没有找到任何内容（found_count=0）
2. LLM API调用失败
3. 摘要生成过程中出错
