# 🔧 数据库连接问题修复指南

## 问题诊断

你遇到的错误：
```
Error: getaddrinfo ENOTFOUND db.tqprqmtlqthryvlzdrco.supabase.co
```

这表示 **Supabase 主机名无法解析**。

---

## 🔍 可能的原因

1. **Supabase 项目不存在或已被删除**
2. **主机名拼写错误**
3. **网络连接问题**
4. **环境变量未正确设置**

---

## ✅ 解决方案

### 步骤 1: 验证 Supabase 项目

1. 访问 https://supabase.com/dashboard
2. 登录你的账号
3. 检查项目是否存在

**如果项目不存在**：
- 需要创建新项目（见下方"创建新项目"）

**如果项目存在**：
- 继续下一步

### 步骤 2: 获取正确的连接信息

1. 在 Supabase Dashboard，选择你的项目
2. 点击左侧 **Settings** → **Database**
3. 找到 **Connection string** 部分
4. 选择 **URI** 模式
5. 复制连接字符串，格式如下：
   ```
   postgresql://postgres:[YOUR-PASSWORD]@aws-0-ap-southeast-1.pooler.supabase.com:5432/postgres
   ```

6. 提取信息：
   ```bash
   DB_HOST=aws-0-ap-southeast-1.pooler.supabase.com  # 从连接字符串中提取
   DB_PASSWORD=[你的密码]       # 创建项目时设置的密码
   ```

### 步骤 3: 测试连接

运行诊断脚本：

```bash
# 设置环境变量
export DB_HOST=aws-0-ap-southeast-1.pooler.supabase.com
export DB_PASSWORD=你的密码

# 运行测试
./scripts/test-supabase-connection.sh
```

### 步骤 4: 运行迁移

如果测试通过，运行迁移：

```bash
cd backend
npm run migrate
```

---

## 🆕 创建新的 Supabase 项目

如果你的项目不存在，需要创建新项目：

### 1. 创建项目

1. 访问 https://supabase.com/dashboard
2. 点击 **"New Project"**
3. 填写信息：
   - **Name**: `financial-compliance-monitoring`
   - **Database Password**: 设置一个强密码（保存好！）
   - **Region**: 选择离你最近的区域（如 Singapore）
4. 点击 **"Create new project"**
5. 等待 2-3 分钟

### 2. 获取连接信息

项目创建完成后：

1. 点击 **Settings** → **Database**
2. 找到 **Connection string**
3. 选择 **URI** 模式
4. 复制连接字符串

### 3. 设置环境变量

```bash
# 从连接字符串中提取
export DB_HOST=aws-0-ap-southeast-1.pooler.supabase.com
export DB_PORT=5432
export DB_NAME=postgres
export DB_USER=postgres
export DB_PASSWORD=你设置的密码
```

### 4. 运行迁移

```bash
cd backend
npm install
npm run migrate
```

## 🧪 手动测试连接

### 使用 psql（如果已安装）

```bash
psql "postgresql://postgres:[密码]@aws-0-ap-southeast-1.pooler.supabase.com:5432/postgres"
```

### 使用 Node.js 测试

创建测试文件 `test-db.js`：

```javascript
const { Pool } = require('pg');

const pool = new Pool({
  host: 'aws-0-ap-southeast-1.pooler.supabase.com',
  port: 5432,
  database: 'postgres',
  user: 'postgres',
  password: '你的密码',
  ssl: { rejectUnauthorized: false }
});

pool.query('SELECT NOW()', (err, res) => {
  if (err) {
    console.error('❌ 连接失败:', err.message);
  } else {
    console.log('✅ 连接成功!', res.rows[0]);
  }
  pool.end();
});
```

运行：
```bash
node test-db.js
```

## 🔧 常见问题

### Q1: DNS 解析失败

**A**: 
1. 检查主机名拼写是否正确
2. 确认 Supabase 项目存在
3. 尝试使用其他网络（如手机热点）

### Q2: 端口无法访问

**A**:
1. 检查防火墙设置
2. 确认 Supabase 项目未暂停
3. 尝试使用 VPN

### Q3: 密码错误

**A**:
1. 在 Supabase Dashboard 重置密码
2. Settings → Database → Reset Database Password
3. 使用新密码重试

### Q4: 连接超时

**A**:
1. 检查网络连接
2. 尝试增加超时时间
3. 检查是否在公司网络（可能有限制）

## 🆘 仍然无法解决？

### 方案 1: 使用本地 PostgreSQL

如果 Supabase 连接有问题，可以先使用本地数据库：

```bash
# 使用 Docker 启动本地 PostgreSQL
docker run -d \
  --name postgres \
  -e POSTGRES_PASSWORD=postgres \
  -p 5432:5432 \
  postgres:14

# 设置环境变量
export DB_HOST=localhost
export DB_PASSWORD=postgres

# 运行迁移
cd backend && npm run migrate
```

### 方案 2: 创建新的 Supabase 项目

如果旧项目有问题，创建一个新项目：

1. 访问 https://supabase.com/dashboard
2. 创建新项目
3. 使用新的连接信息

### 方案 3: 联系 Supabase 支持

如果是 Supabase 服务问题：

1. 访问 https://supabase.com/support
2. 提交支持请求
3. 说明你的项目 ID 和问题
