# 🚀 简化部署方案

## 当前情况

- ✅ Supabase 数据库已创建并迁移完成
- ✅ 代码已推送到 GitHub
- ❌ Render 需要信用卡验证
- ⚠️ Vercel Serverless 有 10 秒超时限制（不适合长任务）

## 📋 推荐方案

### 方案 1：仅部署前端到 Vercel + 本地运行后端（最简单）

**适合**：开发测试、演示

```
Vercel（前端）→ 本地后端 → Supabase（数据库）
```

**优点**：
- 完全免费
- 立即可用
- 前端可以公开访问

**缺点**：
- 后端需要本地运行
- 需要内网穿透（ngrok）让前端访问后端

### 方案 2：Railway（推荐）

**适合**：生产环境

```
Vercel（前端）→ Railway（后端）→ Supabase（数据库）
```

**优点**：
- Railway 提供 $5 免费额度
- 不需要信用卡（初期）
- 支持长时间运行的任务

**缺点**：
- 免费额度用完后需要付费

### 方案 3：Fly.io

**适合**：生产环境

```
Vercel（前端）→ Fly.io（后端）→ Supabase（数据库）
```

**优点**：
- 提供免费额度
- 支持 Docker 部署

**缺点**：
- 可能需要信用卡验证

### 步骤 3：使用 ngrok 暴露本地后端

安装 ngrok：
```bash
brew install ngrok
# 或访问 https://ngrok.com/download
```

运行 ngrok：
```bash
ngrok http 3000
```

ngrok 会给你一个公网 URL，如：
```
https://abc123.ngrok.io
```

### 步骤 4：更新前端配置

在 Vercel Dashboard 中：
1. 进入项目 Settings → Environment Variables
2. 添加：
   ```
   VITE_API_URL=https://abc123.ngrok.io
   ```
3. 重新部署前端

### 步骤 5：访问应用

访问你的 Vercel URL：
```

```

## 🚀 推荐：方案 2（Railway）

如果你想要完整的生产部署，使用 Railway：

### 步骤 1：注册 Railway

访问：https://railway.app

用 GitHub 账号登录（不需要信用卡）。

### 步骤 2：创建新项目

1. 点击 **"New Project"**
2. 选择 **"Deploy from GitHub repo"**
3. 选择 `financial-compliance-monitoring`
4. Railway 会自动检测并部署

### 步骤 3：配置环境变量

在 Railway Dashboard 中：
1. 选择你的服务
2. 点击 **"Variables"**
3. 添加所有环境变量（从 `.render-env.txt` 复制）

### 步骤 4：配置启动命令

在 Railway 中设置：
- **Root Directory**: `backend`
- **Build Command**: `npm install && npm run build`
- **Start Command**: `node dist/index.pgboss.js`

### 步骤 5：部署前端

```bash
cd frontend
npx vercel --prod
```

在 Vercel 环境变量中设置：
```
VITE_API_URL=https://your-app.railway.app
```
