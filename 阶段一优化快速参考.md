## 核心功能

### 1. SimpleRetriever
**用途**: Chunk拆分和BM25检索

**使用示例**:
```typescript
import { SimpleRetriever } from './services/SimpleRetriever';

const retriever = new SimpleRetriever();

// 拆分内容
const chunks = retriever.chunkContent(
  content,
  'https://example.com',
  'main'
);

// 建立索引
retriever.buildIndex(chunks);

// 检索
const results = retriever.retrieve('关键词', 30);
```

### 2. DebugLogger
**用途**: 收集和验证debug信息

**使用示例**:
```typescript
import { DebugLogger } from './services/DebugLogger';

const logger = new DebugLogger('关键词', 'https://example.com');

// 记录各阶段信息
logger.logContentFetch(...);
logger.logChunking(chunks);
logger.logRetrieval(retrievedChunks);
logger.logLLMCall(...);
logger.logLLMAnswer(answer);

// 验证
logger.validate(chunks);

// 获取debug信息
const debugInfo = logger.getDebugInfo();

// 打印到控制台
logger.print();
```

## 故障排查

### 问题1: 测试失败 - natural库错误
**症状**: `SyntaxError: Unexpected token 'export'`

**解决**:
```bash
# 确保jest.config.js存在
# 确保src/__mocks__/natural.ts存在
npm test
```

### 问题2: debugInfo为undefined
**症状**: `result.debugInfo`返回undefined

**检查**:
```bash
# 1. 确认DEBUG_MODE=true
echo $DEBUG_MODE

# 2. 检查.env文件
cat backend/.env | grep DEBUG_MODE

# 3. 重启服务
npm run dev
```

### 问题3: Chunk数量为0
**症状**: 没有生成任何chunks

**原因**: 内容太短，被CHUNK_MIN_SIZE过滤

**解决**:
```bash
# 降低最小长度
CHUNK_MIN_SIZE=50

# 或确保内容足够长（>100字符）
```

## 配置调优

### 调整Chunk大小
```bash
# 更大的chunks（更多上下文，但数量更少）
CHUNK_MAX_SIZE=800
CHUNK_MIN_SIZE=200

# 更小的chunks（更精确，但数量更多）
CHUNK_MAX_SIZE=300
CHUNK_MIN_SIZE=50
```

### 调整检索数量
```bash
# 更多chunks（更全面，但成本更高）
MAX_CHUNKS_PER_KEYWORD=50

# 更少chunks（更快，但可能遗漏信息）
MAX_CHUNKS_PER_KEYWORD=20
```
