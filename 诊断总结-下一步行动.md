### 如果一切正常但未找到关键词

```

- found_count: 0
- not_found_count: 1

- LLM response: {"keywordResults":[{"keyword":"professional","found":false,...}]}
- Keyword matches: [{"keyword":"professional","found":false}]
```

**说明**: 系统正常工作，但LLM判断关键词不存在。

**解决方案**: 
- 手动检查网站是否包含关键词

- 检查LLM提示词是否需要优化

### 如果LLM调用失败

```
SQL结果:
- status: completed
- total_results: 1
- found_count: 0
- not_found_count: 1

Railway日志:
- Error calling LLM API: <error>
- LLM search failed
- Falling back to simple keyword matching
```

**说明**: LLM API调用失败，回退到简单匹配。

**解决方案**:
- 检查Railway环境变量（LLM_API_KEY, LLM_API_URL等）
- 测试LLM API是否可访问
- 查看详细错误信息
