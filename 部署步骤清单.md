## 📋 第 2 步：配置 Supabase 数据库

### 2.1 创建项目

1. 访问 https://supabase.com/dashboard
2. 点击 **"New Project"**
3. 填写：
   - Name: `financial-compliance-monitoring`
   - Password: 设置强密码（**务必保存！**）
   - Region: `Southeast Asia (Singapore)`
4. 等待 2-3 分钟

### 2.2 获取连接信息

1. 项目创建完成后，点击 **Settings** → **Database**
2. 找到 **Connection Pooling** 部分
3. 模式选择 **Transaction**
4. 记录信息：

```
Host: aws-0-ap-southeast-1.pooler.supabase.com
Port: 6543
Database: postgres
User: postgres.[你的项目ID]
Password: [你设置的密码]
```

⚠️ 项目ID在 **Settings** → **General** → **Reference ID**

### 2.3 运行数据库迁移

```bash
# 设置环境变量
export DB_HOST=aws-0-ap-southeast-1.pooler.supabase.com
export DB_PORT=6543
export DB_NAME=postgres
export DB_USER=postgres.你的项目ID
export DB_PASSWORD=你的密码

# 运行迁移
cd backend
npm install
npm run migrate
```

✅ 看到 "Migration completed successfully" 表示成功！

## 📋 第 3 步：部署后端到 Render

### 3.1 创建 Web Service

1. 访问 https://render.com
2. 使用 GitHub 登录
3. 点击 **"New +"** → **"Web Service"**
4. 连接你的 GitHub 仓库

### 3.2 配置服务

```
Name: financial-compliance-backend
Region: Singapore
Branch: main
Root Directory: backend
Runtime: Node
Build Command: npm install && npm run build
Start Command: node dist/index.pgboss.js
Instance Type: Free
```

### 3.3 添加环境变量

点击 **"Advanced"** → **"Add Environment Variable"**

**复制以下内容**（替换实际值）：

```bash
NODE_ENV=production
PORT=3000
TZ=Asia/Shanghai

DB_HOST=aws-0-ap-southeast-1.pooler.supabase.com
DB_PORT=6543
DB_NAME=postgres
DB_USER=postgres.你的项目ID
DB_PASSWORD=你的密码

JWT_SECRET=<第1步生成的密钥>
JWT_EXPIRES_IN=7d

# LLM 配置（根据实际情况选择）
LLM_API_URL=https://office.webullbroker.com/api/oa-ai/open/chat/completions
LLM_API_KEY=dcb0d1c8-82c7-478f-bd9a-7c3023cac4d8
LLM_MODEL=us.anthropic.claude-sonnet-4-20250514-v1:0
LLM_API_KEY_HEADER=authorization
LLM_AUTH_PREFIX=Bearer

JINA_READER_API_URL=https://r.jina.ai
MAX_PARALLEL_WEBSITES=5
RETRIEVAL_TIMEOUT_MS=30000
ENABLE_WEBSITE_ANALYZER=true
DEMO_MODE=false
```

### 3.4 部署

1. 点击 **"Create Web Service"**
2. 等待 5-10 分钟
3. 记录你的后端 URL，例如：
   ```
   https://financial-compliance-backend.onrender.com
   ```

### 3.5 验证

```bash
curl https://your-app.onrender.com/health
```

应该返回：
```json
{
  "status": "healthy",
  "services": {
    "database": "up",
    "queue": "pg-boss"
  }
}
```

## 📋 第 4 步：部署前端到 Vercel

### 4.1 安装 Vercel CLI

```bash
npm install -g vercel
```

### 4.2 配置后端 URL

```bash
cat > frontend/.env.production << EOF
VITE_API_BASE_URL=https://your-app.onrender.com
EOF
```

⚠️ 替换 `your-app.onrender.com` 为第 3 步的 Render URL

### 4.3 部署

```bash
# 登录
vercel login

# 部署
cd frontend
vercel --prod
```

按提示操作：
- Set up and deploy? → **Yes**
- Which scope? → 选择你的账号
- Link to existing project? → **No**
- Project name? → **financial-compliance-frontend**
- In which directory is your code located? → **.**
- Want to override the settings? → **No**

### 4.4 记录前端 URL

例如：
```
https://financial-compliance-frontend.vercel.app
```

## ❓ 常见问题

### Q: 为什么不需要登录？

**A**: 当前版本没有实现用户认证。如需添加认证功能，部署后可以再配置。

### Q: 如何添加认证？

**A**: 部署完成后，我可以帮你添加：
1. 注册/登录 API
2. 登录页面
3. JWT 认证
4. 用户管理

### Q: Render 会休眠吗？

**A**: 是的，免费层 15 分钟无活动后休眠。使用 UptimeRobot 可以防止休眠。

### Q: 数据安全吗？

**A**: 
- 数据存储在 Supabase（加密）
- 建议添加认证保护
- 可以配置 IP 白名单
