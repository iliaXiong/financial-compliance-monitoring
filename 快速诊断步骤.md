# 快速诊断步骤

## 立即执行以下操作

### 1. 在Supabase SQL Editor执行诊断SQL

复制 `诊断test任务.sql` 中的所有SQL语句，粘贴到Supabase SQL Editor并执行。

### 2. 查看Railway日志

打开Railway项目，查看最近的日志，搜索以下关键词：

```
[ContentRetriever] LLM response
[ContentRetriever] LLM search completed
[ContentRetriever] Keyword matches
[TaskScheduler] Step 2: Saving
```

这些日志会显示：
- LLM是否被调用
- LLM返回了什么
- 关键词匹配结果
- 保存了多少条检索结果

### 3. 检查关键信息

#### 从SQL结果中获取：

**执行记录**:
```
status: ?
error_message: ?
duration_seconds: ?
```

**检索结果**（每条记录）:
```
website_url: ?
keyword: ?
found: true/false
content_or_error: ?（如果found=false，这里会显示错误原因）
source_url: ?
```

**统计信息**:
```
total_results: ?
found_count: ?
not_found_count: ?
```

#### 从Railway日志中获取：

搜索 `[ContentRetriever] LLM response`，查看LLM返回的内容（前500字符）

搜索 `[ContentRetriever] Keyword matches`，查看关键词匹配结果

### 4. 常见问题快速检查

#### 问题A: total_results = 0
**说明**: 没有保存任何检索结果
**原因**: 执行过程中出错
**查看**: 执行记录的 error_message 字段

#### 问题B: found_count = 0, not_found_count > 0
**说明**: 有检索结果，但都是 found=false
**原因**: 
1. LLM未找到关键词
2. LLM API调用失败
3. 网站访问失败

**查看**: 
- 检索结果的 content_or_error 字段（会显示具体错误）
- Railway日志中的 LLM 相关日志

#### 问题C: found_count > 0, 但前端显示没有结果
**说明**: 数据库有结果，但前端没有正确显示
**原因**: 
1. 前端缓存了旧的任务ID
2. 前端API调用错误

**解决**: 
1. 清除浏览器缓存
2. 使用正确的任务ID访问

## 预期结果

### 正常情况应该是：

```
执行记录:
- status: completed
- error_message: null
- duration_seconds: 30-120

检索结果统计:
- total_results: 1 (1个网站 × 1个关键词)
- found_count: 0 或 1
- not_found_count: 1 或 0

分析结果:
- summary_count: 1
- comparison_count: 0 (第一次执行)
- cross_site_count: 1
```

### 如果found_count=0:

检索结果详情应该显示：
```
found: false
content_or_error: "Error: ..." 或 "No matches found"
```

这会告诉我们为什么没有找到结果。
