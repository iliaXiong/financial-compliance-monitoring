# 🚀 现在开始部署到 Render + Supabase + Vercel

按照以下步骤完成部署，预计 20-30 分钟。

---

## 📋 第一步：准备 Supabase 数据库（5 分钟）

### 1.1 创建 Supabase 项目

1. 访问 https://supabase.com/dashboard
2. 点击 **"New Project"**
3. 填写信息：
   - **Name**: `financial-compliance-monitoring`
   - **Database Password**: 设置一个强密码（**务必保存！**）
   - **Region**: `Southeast Asia (Singapore)`
4. 点击 **"Create new project"**
5. 等待 2-3 分钟

### 1.2 获取连接信息

项目创建完成后：

1. 点击左侧 **Settings** → **Database**
2. 找到 **Connection Pooling** 部分（不是 Connection string）
3. 模式选择 **Transaction**
4. 复制连接信息：

```
Host: aws-0-ap-southeast-1.pooler.supabase.com
Database: postgres
Port: 6543
User: postgres.[你的项目ID]
Password: [你设置的密码]
```

⚠️ **重要**：
- 使用 **Connection Pooling**（端口 6543）
- 用户名格式是 `postgres.项目ID`（不是 `postgres`）
- 项目 ID 在 Settings → General → Reference ID

### 1.3 运行数据库迁移

在本地运行迁移：

```bash
# 设置环境变量
export DB_HOST=aws-0-ap-southeast-1.pooler.supabase.com
export DB_PORT=6543
export DB_NAME=postgres
export DB_USER=postgres.你的项目ID
export DB_PASSWORD=你的密码

# 运行迁移
cd backend
npm install
npm run migrate
```

✅ 看到 "Migration completed successfully" 表示成功！

---

## 📋 第二步：部署后端到 Render（10 分钟）

### 2.1 创建 Render 账号

1. 访问 https://render.com
2. 使用 GitHub 账号登录

### 2.2 创建 Web Service

1. 点击 **"New +"** → **"Web Service"**
2. 点击 **"Connect a repository"**
3. 授权 Render 访问你的 GitHub
4. 选择你的项目仓库

### 2.3 配置服务

填写以下信息：

```
Name: financial-compliance-backend
Region: Singapore (或其他区域)
Branch: main
Root Directory: backend
Runtime: Node
Build Command: npm install && npm run build
Start Command: node dist/index.pgboss.js
Instance Type: Free
```

⚠️ **关键**：Start Command 必须是 `node dist/index.pgboss.js`

### 2.4 添加环境变量

点击 **"Advanced"** → **"Add Environment Variable"**，添加以下变量：

```bash
# 基础配置
NODE_ENV=production
PORT=3000
TZ=Asia/Shanghai

# 数据库配置（从第一步获取）
DB_HOST=aws-0-ap-southeast-1.pooler.supabase.com
DB_PORT=6543
DB_NAME=postgres
DB_USER=postgres.你的项目ID
DB_PASSWORD=你的密码

# JWT 配置（生成新的密钥）
JWT_SECRET=<运行: openssl rand -base64 32>
JWT_EXPIRES_IN=7d

# LLM 配置（根据你的实际情况）
# 选项 1: 使用 OpenAI
OPENAI_API_KEY=你的OpenAI密钥
OPENAI_MODEL=gpt-4

# 选项 2: 使用自定义 LLM API
LLM_API_URL=https://office.webullbroker.com/api/oa-ai/open/chat/completions
LLM_API_KEY=dcb0d1c8-82c7-478f-bd9a-7c3023cac4d8
LLM_MODEL=us.anthropic.claude-sonnet-4-20250514-v1:0
LLM_API_KEY_HEADER=authorization
LLM_AUTH_PREFIX=Bearer

# Jina Reader 配置
JINA_READER_API_URL=https://r.jina.ai

# 其他配置
MAX_PARALLEL_WEBSITES=5
RETRIEVAL_TIMEOUT_MS=30000
ENABLE_WEBSITE_ANALYZER=true
DEMO_MODE=false
```

### 2.5 生成 JWT Secret

在本地终端运行：

```bash
openssl rand -base64 32
```

复制输出结果，填入 `JWT_SECRET` 环境变量。

### 2.6 部署

1. 点击 **"Create Web Service"**
2. 等待部署完成（约 5-10 分钟）
3. 部署完成后，你会得到一个 URL，例如：
   ```
   https://financial-compliance-backend.onrender.com
   ```

### 2.7 验证部署

在浏览器或终端测试：

```bash
curl https://your-app.onrender.com/health
```

应该返回：
```json
{
  "status": "healthy",
  "services": {
    "database": "up",
    "queue": "pg-boss"
  },
  "timestamp": "2024-..."
}
```

✅ 后端部署完成！**记录下你的后端 URL**

## 📋 第三步：部署前端到 Vercel（5 分钟）

### 3.1 安装 Vercel CLI

```bash
npm install -g vercel
```

### 3.2 创建前端环境变量文件

在项目根目录创建 `frontend/.env.production`：

```bash
cat > frontend/.env.production << EOF
VITE_API_BASE_URL=https://your-app.onrender.com
EOF
```

⚠️ 替换 `your-app.onrender.com` 为你在第二步获得的 Render URL（**不要加 /api 后缀**）

### 3.3 部署到 Vercel

```bash
# 登录 Vercel
vercel login

# 进入前端目录
cd frontend

# 部署到生产环境
vercel --prod
```

按照提示操作：
- **Set up and deploy?** → Yes
- **Which scope?** → 选择你的账号
- **Link to existing project?** → No
- **Project name?** → financial-compliance-frontend
- **In which directory is your code located?** → ./
- **Want to override the settings?** → No

### 3.4 等待部署完成

部署完成后，Vercel 会显示你的前端 URL，例如：
```
https://financial-compliance-frontend.vercel.app
```

### 3.5 验证部署

1. 访问前端 URL
2. 应该能看到登录页面
3. 尝试注册和登录

✅ 前端部署完成！

## 📝 部署后配置（可选）

### 配置自定义域名

**Vercel**:
1. Project Settings → Domains
2. 添加你的域名
3. 配置 DNS

**Render**:
1. Settings → Custom Domain
2. 添加你的域名
3. 配置 DNS

### 防止 Render 休眠

Render 免费层会在 15 分钟无活动后休眠。使用 UptimeRobot 防止休眠：

1. 访问 https://uptimerobot.com
2. 注册账号
3. 添加监控：
   - **Monitor Type**: HTTP(s)
   - **URL**: `https://your-app.onrender.com/health`
   - **Monitoring Interval**: 10 minutes
4. 保存

## 🔧 常见问题

### Q: Render 部署失败？

**A**: 检查以下几点：
1. Start Command 是否正确：`node dist/index.pgboss.js`
2. 环境变量是否都已添加
3. 查看 Render 日志排查错误

### Q: 前端无法连接后端？

**A**: 
1. 检查 `frontend/.env.production` 中的 URL 是否正确
2. 确认后端健康检查通过
3. 检查浏览器控制台是否有 CORS 错误
4. 重新部署前端：`cd frontend && vercel --prod`

### Q: 数据库连接失败？

**A**:
1. 确认使用 Connection Pooler（端口 6543）
2. 确认用户名格式：`postgres.项目ID`
3. 确认密码正确
4. 在 Supabase Dashboard 查看连接状态

### Q: 任务执行失败？

**A**:
1. 检查 LLM API 配置是否正确
2. 查看 Render 日志
3. 确认 Jina Reader API 可访问
