## 3步完成部署

### 步骤1: 在Railway添加环境变量 ⭐

1. 访问 https://railway.app/dashboard
2. 选择项目 `financial-compliance-monitoring`
3. 点击后端服务 → Variables
4. 添加以下5个新变量：

```bash
DEBUG_MODE=true
MAX_CHUNKS_PER_KEYWORD=30
CHUNK_MAX_SIZE=500
CHUNK_MIN_SIZE=100
CHUNK_OVERLAP=50
```

5. 点击保存，Railway会自动重新部署（约3分钟）

### 步骤2: 在Vercel部署前端

**方法A: 如果项目已存在**
1. 访问 https://vercel.com/dashboard
2. 找到项目 `financial-compliance-monitoring`
3. 点击 "Redeploy" 重新部署

**方法B: 如果项目不存在**
1. 访问 https://vercel.com/dashboard
2. 点击 "Add New..." → "Project"
3. 选择 `financial-compliance-monitoring` 仓库
4. 配置：
   - **Root Directory**: `frontend`
   - **Framework**: Vite
   - **Build Command**: `npm run build`
   - **Output Directory**: `dist`
5. 添加环境变量：
   ```
   VITE_API_BASE_URL=https://financial-compliance-monitoring-production.up.railway.app
   ```
6. 点击 "Deploy"

### 步骤3: 验证部署

1. **检查后端**
   ```
   访问: https://financial-compliance-monitoring-production.up.railway.app/health
   ```
   应该看到 `"optimization": {"stage1": "enabled"}`

2. **检查前端**
   ```
   访问你的Vercel URL（例如: https://your-app.vercel.app）
   ```

3. **检查Railway日志**
   - Railway Dashboard → Deployments → 最新部署 → Logs
   - 查找: `[ContentRetriever] Using optimized LLM search`
   - 查找: `DEBUG INFO`

4. **测试功能**
   - 在前端创建测试任务
   - 查看Railway日志中的debug信息
   - 验证Token使用量和成本

## 如果遇到问题

### Railway后端不存在？
按照 `手动部署指南-无需CLI.md` 中的 "如果Railway后端不存在" 部分重新部署

### 前端无法连接后端？
1. 检查Vercel环境变量 `VITE_API_BASE_URL`
2. 确认Railway后端正在运行
3. 查看浏览器控制台错误

### 没有看到DEBUG INFO？
1. 确认Railway环境变量 `DEBUG_MODE=true`
2. 重新部署Railway服务
3. 创建新的测试任务
