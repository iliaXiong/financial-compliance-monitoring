# 数据库连接故障排查指南

## 当前问题

运行迁移时出现以下错误之一：

### 错误 1：Tenant or user not found
```
Migration failed: error: Tenant or user not found
code: 'XX000'
```

**原因**：Reference ID 不正确，或使用了项目名称而不是 Reference ID

### 错误 2：getaddrinfo ENOTFOUND
```
Migration failed: Error: getaddrinfo ENOTFOUND db.xxx.supabase.co
code: 'ENOTFOUND'
```

**原因**：DNS 无法解析主机名，Reference ID 错误或项目未完全创建

## 解决步骤

### 步骤 1：获取正确的 Reference ID

1. 访问 https://supabase.com/dashboard
2. 选择你的项目
3. 点击 **Settings** (左侧齿轮图标)
4. 点击 **General**
5. 找到 "Reference ID"

**Reference ID 特征**：
- ✅ 20 个字符
- ✅ 全部小写字母
- ✅ 示例：`ynbaatdsceqtqwmqhlgu`
- ❌ 不是项目名称（如 "financial-compliance-monitoring"）

### 步骤 2：验证数据库密码

数据库密码是你在创建 Supabase 项目时设置的。

**如果忘记密码**：
1. Settings → Database
2. 点击 "Reset database password"
3. 设置新密码并保存

### 步骤 3：确认项目已完全启动

新创建的 Supabase 项目需要 **2-3 分钟** 才能完全启动。

检查项目状态：
- Dashboard 中项目状态应该是 "Active" (绿色)
- 不应该显示 "Setting up" 或 "Paused"

### 步骤 4：测试连接

运行测试脚本验证连接：

```bash
cd backend
node test-connection.js
```

**期望输出**：
```
Test 1: Connection Pooler (port 6543)
✅ Connection Pooler: SUCCESS
   Database: postgres
   User: postgres

Test 2: Direct Connection (port 5432)
✅ Direct Connection: SUCCESS
   Database: postgres
   User: postgres
```

**如果仍然失败**，检查：
- Reference ID 是否正确（从 Dashboard 复制粘贴）
- 密码是否正确（注意大小写）
- 项目是否已完全启动（等待 3-5 分钟）

### 步骤 5：运行迁移

连接测试成功后，运行迁移：

```bash
./使用直连运行迁移.sh
```

按提示输入：
1. Reference ID（从 Dashboard 复制）
2. 数据库密码

## 常见问题

### Q1: 我确定 Reference ID 和密码都正确，但还是失败

**可能原因**：

1. **项目暂停**
   - 免费项目 7 天不活动会自动暂停
   - 解决：在 Dashboard 中点击 "Restore" 恢复项目

2. **网络问题**
   - 防火墙阻止连接
   - VPN 干扰
   - 解决：尝试关闭 VPN，检查防火墙设置

3. **区域问题**
   - 确认项目区域是 Southeast Asia (Singapore)
   - 主机名应该包含 `ap-southeast-1`

### Q2: Connection Pooler 和 Direct Connection 有什么区别？

| 特性 | Direct Connection | Connection Pooler |
|------|-------------------|-------------------|
| 端口 | 5432 | 6543 |
| 用户名 | `postgres` | `postgres.{reference_id}` |
| 用途 | 本地开发、迁移 | 生产部署 (Render) |
| 连接数 | 有限 | 优化的连接池 |

**重要**：
- ✅ 本地运行迁移：使用 Direct Connection (5432)
- ✅ Render 部署：使用 Connection Pooler (6543)

### Q3: 为什么不能一直使用 Connection Pooler？

Connection Pooler 在某些情况下有限制：
- 不支持某些 PostgreSQL 扩展
- 事务模式下的限制
- 迁移脚本可能需要直接连接

### Q4: 如何确认迁移成功？

成功的迁移会显示：

```
Starting database migrations...
Running migration: 001_create_users_table.sql
✓ Completed migration: 001_create_users_table.sql
Running migration: 002_create_tasks_table.sql
✓ Completed migration: 002_create_tasks_table.sql
...
All migrations completed successfully!
```

然后可以在 Supabase Dashboard 中验证：
1. 点击 **Table Editor**
2. 应该看到所有表：users, tasks, executions, 等

## 下一步：部署到 Render

迁移成功后，继续部署：

### 1. 准备 Render 环境变量

迁移脚本会自动生成 `.render-env.txt` 文件，包含所有需要的环境变量。

### 2. 创建 Render Web Service

1. 访问 https://dashboard.render.com
2. 点击 "New +" → "Web Service"
3. 连接你的 GitHub 仓库
4. 配置：
   - **Name**: financial-compliance-backend
   - **Region**: Singapore
   - **Branch**: main
   - **Root Directory**: backend
   - **Build Command**: `npm install && npm run build`
   - **Start Command**: `node dist/index.pgboss.js`

### 3. 添加环境变量

复制 `.render-env.txt` 中的所有变量到 Render：
- 点击 "Environment" 标签
- 点击 "Add Environment Variable"
- 逐个添加所有变量

**关键变量**：
```
DB_HOST=aws-0-ap-southeast-1.pooler.supabase.com
DB_PORT=6543
DB_USER=postgres.{your_reference_id}
DB_PASSWORD={your_password}
```

### 4. 部署

点击 "Create Web Service"，Render 会自动：
1. 克隆代码
2. 安装依赖
3. 构建应用
4. 启动服务

等待 5-10 分钟完成首次部署。

### 5. 配置 UptimeRobot（可选）

Render 免费版 15 分钟无活动会休眠。使用 UptimeRobot 保持活跃：

1. 访问 https://uptimerobot.com
2. 添加新监控
3. URL: 你的 Render 服务 URL + `/health`
4. 监控间隔: 5 分钟
