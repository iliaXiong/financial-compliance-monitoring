## 可能的原因分析

### 1. 执行未完成
**症状**: 
- 执行状态为 "running"
- 没有 end_time
- 没有检索结果

**原因**:
- 任务执行超时（默认30分钟）
- 服务器重启导致执行中断
- 网络问题导致无法访问目标网站

**解决方案**:
- 检查 Railway 部署日志
- 手动重新执行任务
- 检查目标网站是否可访问

### 2. 执行失败
**症状**:
- 执行状态为 "failed"
- 有 error_message
- 可能有部分检索结果

**原因**:
- SubagentOrchestrator 执行出错
- Jina Reader API 调用失败
- LLM API 调用失败
- 数据库写入失败

**解决方案**:
- 查看 error_message 了解具体错误
- 检查环境变量配置
- 检查 API 密钥是否有效
- 检查网络连接

### 3. 关键词未找到
**症状**:
- 执行状态为 "completed"
- 有检索结果但 found = false
- 没有内容，只有 context 说明

**原因**:
- 目标网站确实不包含这些关键词
- 关键词拼写错误
- 网站内容获取不完整
- 关键词搜索逻辑有问题

**解决方案**:
- 手动访问目标网站确认关键词是否存在
- 尝试使用更通用的关键词
- 检查 Jina Reader 返回的内容
- 调整关键词搜索算法

### 4. 网站访问失败
**症状**:
- 检索结果的 context 包含错误信息
- 如 "Error: ..." 或 "No matches found"

**原因**:
- 网站需要登录或有访问限制
- 网站使用了反爬虫机制
- 网站响应超时
- Jina Reader 无法处理该网站

**解决方案**:
- 检查网站是否公开可访问
- 尝试使用其他网站
- 增加超时时间
- 检查 Jina Reader 的限制

### 5. 分析步骤失败
**症状**:
- 有检索结果且 found = true
- 但没有摘要文档、对比报告或跨站分析

**原因**:
- LLM API 调用失败
- AnalysisService 执行出错
- 数据库写入失败

**解决方案**:
- 检查 LLM API 配置
- 查看后端日志了解具体错误
- 检查数据库连接

## 常见问题

### Q: 为什么所有关键词都未找到？
A: 可能原因：
1. 目标网站确实不包含这些关键词
2. 网站访问失败或内容获取不完整
3. 关键词搜索逻辑有问题
4. Jina Reader 返回的内容不完整

### Q: 为什么执行一直是 running 状态？
A: 可能原因：
1. 执行超时（默认30分钟）
2. 服务器重启导致执行中断
3. 有自动检测机制会将超过30分钟的执行标记为 failed

### Q: 为什么有检索结果但没有摘要？
A: 可能原因：
1. LLM API 调用失败
2. AnalysisService 执行出错
3. 检查后端日志了解具体原因

### Q: 如何重新执行任务？

curl -X POST https://financial-compliance-monitoring-production.up.railway.app/api/tasks/f6762da3-a734-48a1-b000-743c887a9cab/execute
