## 🐛 故障排查

### Render 部署失败？

查看 Render 的 Logs：
1. 在 Render Dashboard 中点击你的服务
2. 点击 **"Logs"** 标签
3. 查看错误信息

常见问题：
- 环境变量缺失或错误
- Build 命令失败（检查 `package.json`）
- Start 命令错误（应该是 `node dist/index.pgboss.js`）

### Vercel 部署失败？

查看 Vercel 的 Deployment Logs：
1. 在 Vercel Dashboard 中点击你的项目
2. 点击失败的部署
3. 查看 Build Logs

常见问题：
- `VITE_API_URL` 环境变量未设置
- Build 命令失败
- Node 版本不兼容

### 前端无法连接后端？

1. 检查 Render 服务是否在运行（状态应该是 "Live"）
2. 检查 `frontend/.env.production` 中的 `VITE_API_URL` 是否正确
3. 在浏览器中访问：`https://YOUR_RENDER_URL.onrender.com/health`
   - 应该返回：`{"status":"ok"}`

### 数据库连接错误？

Render 服务器应该能正常连接 Supabase（支持 IPv6）。

如果仍有问题：
1. 检查 Render 环境变量中的数据库配置
2. 确认使用的是 Connection Pooler 配置
3. 检查 Supabase 项目状态是否为 Active
